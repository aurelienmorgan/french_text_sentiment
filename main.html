Sentiment Analysis in French¶

0. Introduction¶

We will be performing sentiment analysis on Tweeter data from the open dataset by Charles MALAFOSSE which can be found on the following GitHub page. Sentiment is there categorized as either POSITIVE, NEGATIVE, or NEUTRAL.
As we're focusing on NLP Sentiment Classification in French language, we will only consider the 250,000 tweets in French language from that dataset. 
As we will explain in the upcoming sections, we will be training a dual bidirectional-GRU NLP model (@see chapter §4.) using fastText French word embeddings (@see chapter §3.) and transfer learning (@see chapter §2.). It will be first done using review data that we have scrapped from the Web (@see chapter §2.1.), and then again on the French Tweets dataset (@see chapters §1. and §5.2.a).

This will translate in 3 steps as described in the below figure : i) pre-training, ii) warming-up and iii) fine-tuning.
One important note : when training our model following the above-depicted 3-steps, the only difference beween the step 1 on one hand (pre-training) and steps 2 & 3 on the other (warming-up & fine tuning) consists in the model output layer.
It is important to note that the entire data pipeline otherwise remains identical. Neither the pre-processing nor the text tokenization, nor the dimensions of the model input layer (max number of tokens in the imput text) are modified.

As can be observed above, the "NEUTRAL" sentiment is over-represented in the French Tweets dataset with a staggering 45.60%. If a model predicts every tweet to be "NEUTRAL", then that model will have a 45.60% accuracy. This gives high of an incentive for any model trained on such a dataset to be biased towards tagging a tweet as "NEUTRAL".
To avoid that, we will use  downsampling  when training our NLP model, i.e. we will not use all the "NEUTRAL" tweets but randomly pick some of them only. That way, we will train our NLP model on a dataset that is more evenly balanced between the three classes ; even if not perfectly.
One of the adverse direct consequences of downsampling however is that our training dataset will shrink from 219,599 records down to around 50,000.

However still of non-negligible size, this smaller dataset will not allow for the highest predictive power of an NLP model. To circumvent that bereavement, we will resort to  transfer learning . We will first train our model on a different dataset, larger and more generic in nature, and then fine-tune such a model using our French tweet dataset. That way, our model will have gained insight into how to classify french-language text in general and will use that knowledge when starting to learn how to do so on our more specific content.

We will treat this Sentiment Classification problem as a regression problem (we treat the "rating" variable as a continuous variable). That way, our model will be highly penalized if it classifies a "5 stars" product review as a "1 star" product review (even more so than if the matter was dealt with as a pure classification problem) since the objective function will bring to minimizing the distance between the prediction and the real rating (as opposed to simply categorizing a rating in the proper class).

2.2. Custom Text Tokenizer (go back to top ↑)]

We will be using a custom tokenizer for our datasets. It uses a  regex pattern  which has a few specificities to better take advantage of the raw text comments we are encountering in real life.
Most of those specificities have to do with quantities, units, and whether or not the two are space-separated. We deam those specificities important since many times in the French Reviews dataset, prices, volumes, weights and distances do count in the satisfaction one has with regards to a product that is being rated and commented. We want to optimize the chances for our model to grab those relations not less than any other.

3.2. The <UKN> Token (go back to top ↑)]

Below, we calculate the dimensions for the vector associated to the "<UKN>" token. This "<UKN>" token will serve as a placeholder for any word encountered that does not belong to our vocabulary. For instance, when a word is mis-spelled, it probably doesn't belong to our vocabulary. We then associate it to the "<UKN>" ("unknown") token prior to feeding it to our model.
For our "<UKN>" token vector, we take the average of all word embeddings, and we add that vector to our vocab in the last position.
In [21]:
avg_vec = np.zeros( word_to_vec_map[ 'bonjour' ].shape )

    avg_vec += v

avg_vec /= len( word_to_vec_map )

w = "<UNK>"
word_to_vec_map[ w ] = avg_vec ; del avg_vec
word_to_index[ w ] = len( word_to_index )
index_to_word[ len( index_to_word ) ] = w

4. Dual bi-GRU NLP Model (go back to top ↑)]

Compared to Long Short Term Memory networks, the Gated Recurrent Unit model is simpler. It consists of only the update gate and the reset gate. GRU counts one less gate than LSTM, so there are fewer parameters and faster convergence time during training.
Convolutional Neural Network (CNN) and Recurrent Neural Network (RNN) have been widely used in the field of text sentiment analysis and have achieved good results. However, although CNN can extract local information between consecutive words of a sentence, it ignores the contextual semantic information between words. Bidirectional GRU can make up for the shortcomings that CNN can't extract contextual semantic information of long text, but it can't extract the local features of the text as well as CNN. Therefore, we consider a multi-channel model that combines the power of the two.

4.2. Hyperparameters optimization (go back to top ↑)]

To optimize the set of hyperparameters values of our NLP model, we first relate to random search and, in a second time, we add  XGBoost  +  Scikit-Learn  to the mix to get to an even better performing model instance.
Complete details on how it has all been done can be found in this autonomous step-by-step Notebook with full running code .

In short, the whole procedure has taken place on a small subset of the French Reviews training data :
  • First, we trained a bunch of model instances, each using a different set of randomly-generated hyperparameter values.
  • Second, with this information, we trained an XGBoost regressor to predict NLP model performance based on the set of hyperparameters values
  • Third, we feed the XGBoost regressor with tens millions of sets of randomly-generated hyperparameters values and select the ones predicted to bring the best performance for our NLP model architecture.
The first step gave us one NLP model instance which we call the "best random search" model instance. The third step gave us one NLP model instance which we call the "best XGBoost-identified" model instance.
We then fitted few most promissing sets of hyperparameters values on the entire training dataset, and this finally gave us one NLP model instance which we call the "best refitted" model instance.

5.2. Warming-Up (go back to top ↑)]

5.2.a. Training / Validation / Holdout data subsetting (go back to top ↑)]

In the pre-training phase (§5.1. above), transfer Learning took place on the French Reviews dataset. The warming-up phase here, on the other hand, takes place on our target French Tweets dataset. As seen in §1.3. , this dataset is made up of three sentiment classes which are "NEGATIVE", "NEUTRAL" and "POSITIVE".
In [34]:

If we recall from §1.3. above, the French Tweets dataset being highly imbalanced, it needs downsampling to address the fact that the "NEUTRAL" sentiment class is way over-represented. But prior to doing that, in order to evaluate fairly the performance of our model, we need splitting our real-life dataset into training / validation / holdout subsets.

This operation has to take place now, since we indeed want to respect the elementary rules of model training which are :
  • we measure the final performance of a model on :
    1. data it had never been presented before (the holdout subset),
    2. data drawn from the real life distribution it will have to predict from when deployed into production (no alteration of the resampling kind),
  • the validation and holdout datasets MUST be drawn from identical distributions,
  • it is acceptable to train a model on manipulated data (using techinques such as oversampling/undersampling or data augmentation)

The whole idea behind warming-up consists in bootstrapping the transfer learning. We do that by assigning weights to the new output layer that are more optimum than the initial random values. By freezing all the preceeding layers, the optimizer focuses and these last weights alone, which gives them a huge advantage when later actually operating the model fine-tuning in the upcoming section.

6. Conclusion (go back to top ↑)]

NLP models on languages others than English are seldom. And it's mostly because labelled data in those languages is hard to come by.
The project of the herein notebook being Sentiment Classification on French Texts, we've covered many aspects of techniques that can be handy when working on real-world data which can often be imperfect and/or costly to acquire :
  • we've seen the power of transfer learning from web-scrapped, rather generic, French Reviews to domain-specific French Tweets (specific to soccer in our use case).
  • we've also used XGBoost as a supercharger to our hyperparameters optimization (in this separate dedicated notebook )
  • we've also covered means at our disposal when dealing with highly imbalanced datasets
The above-depicted technical "tricks" in our toolset allowed to train a dual bi-GRU model architecture in an optimum fashion for such a model to tackle efficiently some everyday-life problems.
Ours here is domain-specific to soccer, but any public or private organization could "tune" our model, for instance on a brand-specific dataset (which it would posess internally), by simply following §5.2. & §5.3., which is really quick. Such an NLP model would for instance be a well suited technical solution for anyone willing to implement an automatic Twitter monitoring system for a brand operating on French-speaking markets. This would allow to identify any potential "bad buzz" on the social media platform around the brand as-soon-as any ever occurs. On the other hand, being notified on POSITIVE French Tweets would for instance allow to implement a targetted promotional campaign involving enthousiastic/supportive customers and thus build a resilient community around the brand.
